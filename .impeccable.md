## Design Context

### Users
Mix of sysadmins managing multiple Linux servers and small business owners setting up their own backups. Both technical and non-technical users. Context: checking backup status, configuring sources/destinations, troubleshooting failures. Often accessed via browser on desktop, occasionally on mobile/tablet.

### Brand Personality
**Reliable, Professional, Trustworthy.** The interface should evoke confidence and trust — "my backups are safe." Not flashy, not playful. Like a well-built tool that gets out of the way.

### Aesthetic Direction
- **Theme**: Keep current DaisyUI dark theme as default. Polish rough edges, don't redesign.
- **Visual tone**: Utilitarian with warmth. Dense enough for sysadmins, clear enough for non-technical users.
- **Logo**: Orange server/stack icon with "GNIZA" in bold letter-spacing, "Linux Backup Manager" subtitle.
- **Colors**: DaisyUI theme variables only. Orange accent from logo. Green = success, Red = error, Orange = warning.
- **Anti-references**: Don't make it look like a consumer SaaS product. Don't add gratuitous animations or illustrations. No gradient text, glassmorphism, or "modern startup" aesthetics.
- **References**: Proxmox VE dashboard (information density), Gitea (clean DaisyUI-like feel), Grafana (dark theme done well).

### Design Principles
1. **Trust through clarity** — Every screen should answer "are my backups working?" within 2 seconds.
2. **Dense but not cramped** — Show information efficiently. Tables over cards. Data over decoration.
3. **Progressive disclosure** — Start simple, reveal complexity through interaction. Basic options first, advanced behind expandable sections.
4. **Consistent and predictable** — Same patterns everywhere. Same button sizes, same spacing, same color meanings.
5. **Accessible by default** — WCAG AA compliance. Keyboard navigable. Screen reader friendly. Status communicated through text + color, never color alone.

### Technical Stack
- Flask + Jinja2 templates
- DaisyUI 5.x + Tailwind CSS (browser JIT, no build step)
- Alpine.js for interactivity
- HTMX for server-driven updates
- 19 DaisyUI themes available, dark default
