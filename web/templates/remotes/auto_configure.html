Step 1: Run on Remote Server

SSH into the server you want to use as a backup destination and run this command as root:

curl -fsSL https://git.linux-hosting.co.il/shukivaknin/gniza4linux/raw/branch/main/scripts/setup-remote.sh | sudo bash -s -- --destination

Alternative: wget -qO- https://git.linux-hosting.co.il/shukivaknin/gniza4linux/raw/branch/main/scripts/setup-remote.sh | sudo bash -s -- --destination

The script will install dependencies, create a backup user, generate SSH keys, and display a croc code. Enter that code below.

Step 2: Receive Configuration

Enter the croc code displayed by the remote setup script and a name for this destination.

Direct SSH Setup

Provide SSH credentials and GNIZA will automatically connect to the remote server, install dependencies, create a backup user, and configure it as a destination.

Auto-Configure Destination

Automatically configure an SSH backup destination.

Via Remote Script

  1. Run the setup script on the remote server (installs rsync, creates backup user, generates SSH keys).
  2. The script displays a one-time croc code.
  3. Enter the code here to securely receive the SSH key and server details.
  4. GNIZA saves the destination config and tests the connection.

Direct SSH Setup

  1. Provide SSH credentials for the remote server (root or user with passwordless sudo).
  2. GNIZA connects via SSH, uploads the setup script, and runs it automatically.
  3. The destination is configured and the connection is tested.

Requirements

  • Via Remote Script: croc must be installed on both servers. Both need internet access.
  • Direct SSH: SSH access to the remote server (root or passwordless sudo). sshpass required for password auth.
