2. Install Required Packages

                    
On the destination server:

                    

                        
apt-get install ssh sudo rsync

                        
                    

                


                


                
                

                    
3. Create a Backup User

                    
Create a dedicated user for backups:

                    

                        
adduser --disabled-password --gecos "GNIZA backup user" gniza

                        
                    

                


                


                
                

                    
4. Configure Sudoers

                    
Allow the backup user to run rsync, mkdir, and chown with elevated privileges:

                    

                        
echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /bin/mkdir, /bin/chown" >> /etc/sudoers

                        
                    

                    
With MySQL:

                    

                        
echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /bin/mkdir, /bin/chown, /usr/bin/mysqldump, /usr/bin/mysql" >> /etc/sudoers

                        
                    

                    
With PostgreSQL:

                    

                        
echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /bin/mkdir, /bin/chown, /usr/bin/pg_dump, /usr/bin/pg_dumpall, /usr/bin/psql" >> /etc/sudoers

                        
                    

                


                


                
                

                    
5. Set Up SSH Access

                    
On the destination server, set up SSH authorized keys for the backup user:

                    

                        
mkdir -p /home/gniza/.ssh
chown gniza:gniza /home/gniza/.ssh
chmod 700 /home/gniza/.ssh
echo "" > /home/gniza/.ssh/authorized_keys
chown gniza:gniza /home/gniza/.ssh/authorized_keys
chmod 600 /home/gniza/.ssh/authorized_keys
