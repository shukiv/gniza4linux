Step 1: Run on Remote Server

SSH into the server you want to back up and run this command as root:

Alternative: wget -qO- https://git.linux-hosting.co.il/shukivaknin/gniza4linux/raw/branch/main/scripts/setup-remote.sh | sudo bash -s -- --source

Auto-Configure Source

Automatically configure a remote SSH backup source using croc for secure key exchange.

How it works

  1. Run the setup script on the remote server (installs rsync, creates backup user, generates SSH keys).
  2. The script displays a one-time croc code.
  3. Enter the code here along with which folders to back up.
  4. GNIZA saves the source config and tests the SSH connection.

Requirements

  • croc must be installed on both this server and the remote.
  • Both servers must have internet access (croc uses a relay for NAT traversal).
