2. Install Required Packages

                            
On the source server:

                            

                                
apt-get install ssh sudo rsync

                                
                            

                        


                        


                        
                        

                            
3. Create a Backup User

                            
Create a dedicated user for backups:

                            

                                
adduser --disabled-password --gecos "GNIZA backup user" gniza

                                
                            

                        


                        


                        
                        

                            
4. Configure Sudoers

                            
Allow the backup user to run rsync with elevated privileges (needed for reading all files):

                            

                                
echo "gniza ALL = NOPASSWD: /usr/bin/rsync" >> /etc/sudoers

                                
                            

                            
With MySQL:

                            

                                
echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /usr/bin/mysqldump, /usr/bin/mysql" >> /etc/sudoers

                                
                            

                            
With PostgreSQL:

                            

                                
echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /usr/bin/pg_dump, /usr/bin/pg_dumpall, /usr/bin/psql" >> /etc/sudoers

                                
                            

                        


                        


                        
                        

                            
5. Set Up SSH Access

                            
On the source server, set up SSH authorized keys for the backup user:

                            

                                
mkdir -p /home/gniza/.ssh
chown gniza:gniza /home/gniza/.ssh
chmod 700 /home/gniza/.ssh
echo "" > /home/gniza/.ssh/authorized_keys
chown gniza:gniza /home/gniza/.ssh/authorized_keys
chmod 600 /home/gniza/.ssh/authorized_keys

Source Editor


Add or edit a backup source configuration.


Fields


    
    
  • Name — Unique identifier (letters, digits, dash, underscore; max 32 chars).
    • 
    
  • Folders — Comma-separated paths to back up. Use Browse to pick folders interactively.
    • 
    
  • Include/Exclude — Glob patterns to filter files (e.g. *.conf, *.log).
    • 
    
  • Pre/Post hooks — Shell commands to run before/after the backup.
    • 



Source types


    
    
  • Local — Default. Back up folders on this machine.
    • 
    
  • SSH — Pull files from a source server via SSH/rsync.
    • 
    
  • Cloud — Pull files from S3, Google Drive, or any rclone-supported remote.
    • 



When using a source server, specify remote paths in the Folders field instead of local paths.


MySQL


Enable MySQL to dump databases alongside files. Choose 'All databases' or select specific ones. Leave user/password empty for socket auth.


PostgreSQL


Enable PostgreSQL to dump databases alongside files. Uses pg_dump for each database and pg_dumpall for roles. Leave user/password empty for peer auth.


Crontab


Enable Crontab to back up user crontabs, /etc/crontab, and /etc/cron.d/ files. Specify which users' crontabs to dump (comma-separated, default: root).
