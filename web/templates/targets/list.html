2. Install Required Packages

On the source server:

apt-get install ssh sudo rsync

3. Create a Backup User

Create a dedicated user for backups:

adduser --disabled-password --gecos "GNIZA backup user" gniza

4. Configure Sudoers

Allow the backup user to run rsync with elevated privileges (needed for reading all files):

echo "gniza ALL = NOPASSWD: /usr/bin/rsync" >> /etc/sudoers

With MySQL:

echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /usr/bin/mysqldump, /usr/bin/mysql" >> /etc/sudoers

With PostgreSQL:

echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /usr/bin/pg_dump, /usr/bin/pg_dumpall, /usr/bin/psql" >> /etc/sudoers

5. Set Up SSH Access

On the source server, set up SSH authorized keys for the backup user:

mkdir -p /home/gniza/.ssh
chown gniza:gniza /home/gniza/.ssh
chmod 700 /home/gniza/.ssh
echo "" > /home/gniza/.ssh/authorized_keys
chown gniza:gniza /home/gniza/.ssh/authorized_keys
chmod 600 /home/gniza/.ssh/authorized_keys

Sources

List and manage backup sources. A source defines what to back up: folders, file patterns, and optionally MySQL databases.

Table columns

  • Name — Unique source identifier.
  • Folders — Comma-separated paths to back up.
  • Enabled — Whether the source is active.

Tips

  • You need at least one source and one destination before you can run backups.
  • Disable a source to skip it during 'Backup All' without deleting it.
