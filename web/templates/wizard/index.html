Prepare Your Destination Server

Before configuring GNIZA, you need to prepare the server that will store your backups. Follow the steps below on your destination server.

1. Your SSH Key

GNIZA uses SSH key authentication to connect to the destination server. You'll need to copy your public key to the server later.

Key generated! Copy the public key above and use it in step 4 below.

2. Install Required Packages

On the destination server, install the required packages:

apt-get install ssh sudo rsync

3. Create a Backup User

Create a dedicated user for backups on the destination server:

adduser --disabled-password --gecos "GNIZA backup user" gniza

4. Configure Sudoers

Allow the backup user to run rsync with elevated privileges (needed for preserving file ownership). If you'll be backing up MySQL databases, include mysqldump too:

echo "gniza ALL = NOPASSWD: /usr/bin/rsync" >> /etc/sudoers

If MySQL backup is needed, use instead:

echo "gniza ALL = NOPASSWD: /usr/bin/rsync, /usr/bin/mysqldump" >> /etc/sudoers

5. Set Up SSH Access

On the destination server, set up the SSH authorized keys for the backup user. Replace <your-ssh-key> with the public key from step 1 above:

mkdir -p /home/gniza/.ssh
chown gniza:gniza /home/gniza/.ssh
chmod 700 /home/gniza/.ssh
echo "" > /home/gniza/.ssh/authorized_keys
chown gniza:gniza /home/gniza/.ssh/authorized_keys
chmod 600 /home/gniza/.ssh/authorized_keys
